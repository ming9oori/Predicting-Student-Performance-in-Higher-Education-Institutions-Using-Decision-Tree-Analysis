## ⚠️ 주의사항 및 한계점 설명

### Data Leakage 이슈
발표 시 반드시 언급:

```
한계점:
현재 모델에서 'Failed' 변수를 만들 때 사용한 Q12(낙제 과목 수)가
그대로 예측 특성으로 포함되어 있어 순환 논리가 발생했습니다.

실제 활용 시에는:
- Q12를 특성에서 제거
- 다른 예측 변수들만 사용
- 또는 Q12를 다른 방식으로 인코딩

이것은 중요한 학습 경험이며,
실제 데이터 과학에서 흔히 발생하는 문제입니다.
```

### 합성 데이터 사용
```
데이터 한계:
- 실제 학생 데이터가 아닌 합성 데이터 사용
- 실제 환경에서는 다른 결과 가능
- 향후 실제 데이터로 검증 필요
```

## 📝 질문 대비

### 예상 질문과 답변

**Q1: "왜 이 세 가지 알고리즘을 선택했나요?"**
```
A: 논문에서 사용한 알고리즘을 재현하기 위해서입니다.
   J48, Random Tree, REPTree는 모두 의사결정 트리 계열로,
   각각 다른 특성(정보 이득, 무작위 분할, 가지치기)을 가지고 있어
   비교 분석에 적합합니다.
```

**Q2: "실제로 이 모델을 사용하려면?"**
```
A: 몇 가지 개선이 필요합니다:
   1. 실제 학생 데이터 수집
   2. Data leakage 문제 해결 (Q12 제거)
   3. 개인정보 보호 정책 수립
   4. 정기적 모델 업데이트
   5. 윤리적 사용 가이드라인
```

**Q3: "정확도가 100%인데 문제가 있나요?"**
```
A: 네, 오히려 문제입니다. 이는 data leakage 때문입니다.
   목표 변수(Failed)를 만들 때 사용한 특성(Q12)이
   그대로 예측에 사용되어 완벽한 예측이 가능해졌습니다.
   실제로는 Q12를 제거하고 다른 특성만으로 예측해야 합니다.
```

**Q4: "합성 데이터와 실제 데이터의 차이는?"**
```
A: 합성 데이터는 논문의 구조를 따라 생성한 것으로,
   실제 학생들의 복잡한 패턴을 완전히 반영하지 못합니다.
   실제 데이터에서는:
   - 더 복잡한 관계
   - 예측하기 어려운 패턴
   - 낮은 정확도(하지만 여전히 유용)
```

**Q5: "이 연구의 실용적 가치는?"**
```
A: 세 가지 측면에서 가치가 있습니다:
   1. 학문적: 논문 재현 연구의 중요성
   2. 기술적: 데이터 과학 파이프라인 구축 경험
   3. 실용적: 실제 교육 현장에 적용 가능한 프레임워크
```
